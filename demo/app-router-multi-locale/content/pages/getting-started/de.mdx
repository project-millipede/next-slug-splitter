# Erste Schritte

Willkommen in der **next-slug-splitter** Demo. Dies ist eine _leichte_ Seite —
sie enthält nur normale Markdown-Elemente und keine eigenen React-Komponenten.

## Warum ist das wichtig?

In einer typischen Next.js-Content-Seite teilen sich alle Seiten einer
dynamischen Route (z. B. `[...slug].tsx`) dasselbe JavaScript — inklusive
schwerer Komponenten, die nur wenige Seiten wirklich verwenden.

**next-slug-splitter** löst das so:

1. MDX-Inhalte werden beim Build analysiert
2. Seiten mit eigenen React-Komponenten werden erkannt
3. Für diese "schweren" Seiten werden eigene Route Handler generiert
4. Die gemeinsame Catch-all-Route bleibt für "leichte" Seiten schlank

## Ergebnis

- Leichte Seiten liefern minimales JavaScript aus
- Schwere Seiten bündeln nur die Komponenten, die sie brauchen
- Kein manuelles Route-Management nötig
