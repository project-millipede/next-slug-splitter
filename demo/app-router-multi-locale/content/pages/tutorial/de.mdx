# Tutorial

Dies ist eine weitere _leichte_ Seite. Sie nutzt normales Markdown und eine
leichte Komponente aus dem MDX-Komponenten-Scope.

<Callout>
Dieser Callout wird während der Analyse erkannt, aber nicht in generierte
Handler importiert, weil er bereits im MDX-Komponenten-Scope verfügbar ist.
</Callout>

## Projekt einrichten

So integrierst du **next-slug-splitter** in dein Next.js-Projekt:

### 1. Paket installieren

```bash
pnpm add next-slug-splitter
```

### 2. Next.js-Konfiguration wrappen

```typescript
import { withSlugSplitter } from 'next-slug-splitter/next';

export default withSlugSplitter(nextConfig, {
  routeHandlersConfig
});
```

### 3. Targets definieren

Jedes Target beschreibt einen Content-Bereich (z. B. docs oder blog) mit einer
eigenen Catch-all-Route und einem Content-Verzeichnis.

### 4. Normalen Build starten

```bash
next build
```

> Der Next-Adapter analysiert die MDX-Inhalte und generiert optimierte Route
> Handler automatisch während des Builds. Die eigenständige CLI ist optional.

## Was passiert beim Build?

Die Bibliothek generiert `generated-handlers/` Dateien neben deiner
Catch-all-Route. Diese Dateien sind gitignored und werden bei jedem Build neu
erstellt.
